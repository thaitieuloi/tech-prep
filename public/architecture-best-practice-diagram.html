RBAC Request Flow — POST /api/orders (CUSTOMER role)

1
Client gets JWT from Keycloak
POST /realms/ecommerce/protocol/openid-connect/token with credentials. Keycloak returns JWT containing realm_access.roles: ["CUSTOMER"].
2
Kong — Layer 1 JWT validation
POST /api/orders + Authorization: Bearer JWT. Kong JWT plugin calls Keycloak JWKS endpoint, verifies signature and expiry. Passes if valid (does NOT check roles at this layer). Returns 401 if invalid.
3
Order Service — Layer 2 RBAC
KeycloakJwtAuthConverter extracts realm_access.roles from JWT and maps to Spring ROLE_CUSTOMER. SecurityConfig checks hasAnyRole('CUSTOMER','VENDOR','ADMIN','SUPER_ADMIN') → passes. Controller calls service.
4
Order-Service (A) calls Product-Service (B)
Feign HTTP. Product-Service also validates JWT (defense in depth). reduceStock endpoint requires authenticated() — any valid token passes.
5
RBAC Ownership — Service Layer
Order saved with customerId = jwt.getSubject(). When CUSTOMER calls GET /api/orders/{id}, service checks order.customerId == requesterId. Returns 403 Forbidden if they try to access another customer's order.
6
Kafka async — no RBAC needed
order-created published. Notification-Service consumes from internal Kafka topic — no JWT, no RBAC, fully internal event-driven.
7
Observability — RBAC audit trail
Logs include traceId, service, and user role. Kibana dashboard can filter denied requests (403) by role for security audit.

Test RBAC Quick Commands

# Get token for each role
get_token() {
  curl -s -X POST http://localhost:8180/realms/ecommerce/protocol/openid-connect/token \
    -d "client_id=ecommerce-frontend&username=$1&password=$2&grant_type=password" \
    | python3 -m json.tool | grep access_token
}

get_token superadmin super123    # SUPER_ADMIN
get_token admin1     admin123    # ADMIN
get_token vendor1    vendor123   # VENDOR
get_token customer1  customer123 # CUSTOMER

# Test: CUSTOMER can read products
curl http://localhost:8000/api/products -H "Authorization: Bearer $CUSTOMER_TOKEN"
# → 200 OK

# Test: CUSTOMER cannot create product → 403
curl -X POST http://localhost:8000/api/products \
  -H "Authorization: Bearer $CUSTOMER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"Test","price":10.0,"stock":100,"category":"Electronics"}'
# → 403 Forbidden

# Test: VENDOR can create product
curl -X POST http://localhost:8000/api/products \
  -H "Authorization: Bearer $VENDOR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"My Product","price":99.99,"stock":50,"category":"Books"}'
# → 201 Created (vendorId set from JWT subject)

# Test: VENDOR cannot delete → 403
curl -X DELETE http://localhost:8000/api/products/{id} \
  -H "Authorization: Bearer $VENDOR_TOKEN"
# → 403 Forbidden

# Test: ADMIN can view all orders
curl http://localhost:8000/api/orders -H "Authorization: Bearer $ADMIN_TOKEN"
# → 200 OK with all orders

# Test: CUSTOMER can only see own orders
curl http://localhost:8000/api/orders/{other_customer_order_id} \
  -H "Authorization: Bearer $CUSTOMER_TOKEN"
# → 403 Forbidden

Technology Stack

RBAC — 2-Layer Enforcement
Layer 1: Kong JWT plugin (401 if invalid). Layer 2: Spring @PreAuthorize + KeycloakJwtAuthConverter + ownership checks in service
Keycloak 24 — 4 Roles
SUPER_ADMIN / ADMIN / VENDOR / CUSTOMER. realm_access.roles in JWT. ProtocolMapper auto-includes roles in access token
KeycloakJwtAuthConverter
Extracts realm_access.roles. Filters Keycloak internals (default-roles, offline_access). Maps to ROLE_XXX GrantedAuthority
Kong 3.6 (Sole Gateway)
JWT plugin validates signature via JWKS. Does not enforce roles — defers to services. Rate limiting, CORS, Zipkin
Product Service RBAC
SecurityConfig URL rules + @PreAuthorize(Roles.IS_VENDOR_OR_ABOVE). VendorId ownership: VENDOR can only edit own products
Order Service RBAC
Ownership check: customerId == jwt.getSubject(). @PreAuthorize IS_ADMIN_OR_SUPER for admin ops. IS_SUPER_ADMIN for delete
Product Service
Spring Boot 3.2. REST + GraphQL. vendorId on Product entity. @Cacheable Redis. Kafka Producer. Gradle
Order Service
OpenFeign to Product-Service. Resilience4j CB. Redis Cache. Kafka Producer. updateOrderStatus endpoint
Notification Service
Kafka Consumer 3 topics. No RBAC needed — internal. Spring Mail. MailHog SMTP
PostgreSQL 16
product_db (+ vendor_id column). order_db. notification_db. keycloak_db. Port 5432
Redis 7
Product cache TTL 10 min. Order cache by customerId TTL 5 min. LRU 256MB
Apache Kafka 7.6
3 topics. order-created, order-status-changed, product-created. Kafka UI :9080
Resilience4j
Circuit Breaker on Feign. Fallback to PENDING order. SlidingWindow 10. 50% threshold
Zipkin 3
Traces include role context for audit. Micrometer Brave. Kong plugin. Port 9411
ELK Stack 8.13
Logstash parses role from JSON logs. Kibana filter 403 errors by role for security audit
Docker Compose + .env
17 containers. .env for runtime config. Health checks. Named volumes. Bridge network
